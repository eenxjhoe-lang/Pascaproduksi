📋 Langkah 1 — Siapkan Google Sheets

1
Buat atau gunakan Google Sheets yang sudah ada
Pastikan Sheets memiliki 3 sheet dengan nama mengandung kata kunci:
KAKAP atau SUNGAI KAKAP → PP. Sungai Kakap
SUKABANGUN → PP. Sukabangun
JELAI atau KUALA JELAI → PP. Kuala Jelai
2
Pastikan format kolom sesuai DOPK
Baris 1 = header, data mulai baris 2
Kolom ExcelIndeksIsiContoh
A0NO (angka urut — wajib)1, 2, 3, ...
B1BULANJANUARI, FEBRUARI, ...
D3NAMA KAPALKeprindo 8
H7PEMILIKAlimin
N13TRIP PENANGKAPAN (HARI)11
O14JUMLAH PRODUKSI (KG)2055
R17NOMINAL BAYAR PNBP (RP)847450
S18KETERANGANLUNAS
💡 Kolom C, E, F, G, I, J, K, L, M, P, Q bisa berisi apa saja (data kapal, tanggal, kode biling, dll). Dashboard hanya membaca kolom yang tertera di atas.

⚡ Langkah 2 — Deploy Google Apps Script

1
Buka Google Apps Script dari Sheets Anda
Di Google Sheets: klik menu Extensions → Apps Script
2
Tempel kode dari apps-script.gs
Hapus semua kode default di editor, lalu paste isi file apps-script.gs
3
Klik Save (ikon disket atau Ctrl+S)
Beri nama project, misalnya: DOPK-Dashboard-API
4
Deploy sebagai Web App
Klik Deploy → New deployment
▸ Type: Web app
▸ Execute as: Me
▸ Who has access: Anyone ← PENTING
Klik Deploy
5
Salin URL Web App yang muncul
Formatnya seperti ini: https://script.google.com/macros/s/AKfycb.../exec
URL ini adalah "alamat API" yang akan dipakai oleh dashboard.
⚠️ Setiap kali Anda mengubah kode apps-script.gs, Anda harus membuat New deployment baru dan mendapatkan URL baru.

⚙️ Langkah 3 — Konfigurasi Dashboard

Opsi A — Cara Cepat (via browser)

1
Buka dashboard di browser
Jalankan via server lokal atau hosting
2
Klik tombol "⚙️ Hubungkan Spreadsheet" di topbar
Modal konfigurasi akan terbuka
3
Paste URL Apps Script, klik Test, lalu Simpan
Dashboard langsung memuat data. URL disimpan di browser.

Opsi B — Cara Permanen (via data.json)

Edit file data.json agar URL tersedia bawaan untuk semua pengunjung:

// data.json
"spreadsheet": {
  "api_url":       "https://script.google.com/macros/s/AKfycb.../exec",
  "refresh_menit": 5,
  "auto_fetch":    true
}
✅ Dengan Opsi B, siapa saja yang mengakses URL dashboard Anda akan langsung melihat data terbaru tanpa perlu konfigurasi apapun.

🌐 Langkah 4 — Hosting (Agar Bisa Diakses Semua Orang)

Pilih salah satu metode hosting sesuai kebutuhan:

🆓 GitHub Pages (Gratis)
1. Upload semua file ke repository GitHub
2. Settings → Pages → Branch: main
3. Dashboard tersedia di: username.github.io/repo
✓ Gratis, mudah, HTTPS otomatis
⚡ Netlify / Vercel (Gratis)
1. Drag & drop folder ke netlify.com atau vercel.com
2. Dapatkan URL langsung
3. Auto-deploy jika folder diupdate
✓ Paling cepat, custom domain
🖥️ Server Internal KKP
1. Copy semua file ke folder webserver
2. Akses via IP/domain intranet
3. Tidak perlu koneksi internet untuk dashboard
✓ Privat, kontrol penuh
💻 Lokal (Satu Komputer)
Jalankan di terminal:
python -m http.server 8080
Buka: http://localhost:8080
⚠ Hanya bisa diakses dari komputer yang sama

❓ Pertanyaan Umum

Dashboard tidak memuat data, padahal URL sudah benar
Pastikan saat deploy Apps Script, "Who has access" = Anyone. Jika diset ke "Anyone with Google Account", perlu login. Coba juga buka URL Apps Script langsung di browser — harus menampilkan JSON.
Data tidak update meski sudah diisi di Sheets
Dashboard refresh otomatis setiap 5 menit. Klik tombol Refresh di topbar untuk memperbarui seketika. Pastikan baris baru di Sheets memiliki angka di kolom A (NO).
Sheet tidak terdeteksi oleh Apps Script
Nama sheet harus mengandung salah satu dari: KAKAP, SUNGAI KAKAP, SUKABANGUN, JELAI, KUALA JELAI. Nama tidak peka huruf besar/kecil.
URL Apps Script berubah setelah update kode
Setiap "New deployment" menghasilkan URL baru. Update data.json → spreadsheet.api_url dengan URL baru tersebut. Alternatif: gunakan "Manage deployments" untuk update deployment yang ada tanpa mengubah URL.
Apakah data aman?
Apps Script membaca Sheets dengan hak akses akun Google Anda. Data diekspos hanya melalui URL API yang Anda buat. Jika Sheets bersifat privat, hanya API-nya yang publik — isi Sheets tidak bisa diakses langsung. Untuk keamanan lebih tinggi, gunakan "Anyone with Google Account" dan tambahkan autentikasi token di Apps Script.
